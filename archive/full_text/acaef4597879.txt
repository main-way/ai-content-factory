Anthropic за два дня выкатила сразу два обновления, которые касаются ИИ-агента для написания кода Claude Code. 18 июня компания добавила артефакты: теперь агент умеет превращать ход работы над задачей в живую веб-страницу, которую можно открыть в браузере и показать команде. Днем ранее вышел крупный апдейт Claude Design, который лучше связал дизайн-инструмент с Claude Code.
Артефакт в Claude Code — это веб-страница, которую агент собирает из полного контекста сессии: кода в репозитории, подключенных коннекторов и самого диалога. Это может быть разбор пул-реквеста, дашборд с фильтрами и сортировкой, страница разбора инцидента или релизный чек-лист, который заполняется сам по мере выполнения работы. Поднимать источники данных и инфраструктуру не нужно — агент строит страницу из того, что уже есть.
Страница живет по постоянной ссылке и обновляется на месте: когда агент дописывает работу, открытая у коллег вкладка подхватывает новую версию, а история версий позволяет откатиться к любой из предыдущих. По умолчанию артефакт виден только автору, поделиться им можно внутри своей организации, публичной ссылки нет. Доступность — бета для планов Team и Enterprise, из CLI и десктоп-приложения, а смотреть результат можно в любом браузере.
Вторая часть — связка с Claude Design, которая теперь работает в обе стороны. Команда /design-sync подтягивает дизайн-систему из локальной базы кода, чтобы прототип сразу начинался с реальных компонентов. Когда дизайн готов, его отдают обратно в Claude Code, который продолжает с уже существующей работы, а не с нуля по скриншоту. А команда /design позволяет создавать и синхронизировать дизайн-проекты, не выходя из терминала. В сам апдейт Claude Design также вошли переделанный импорт дизайн-систем из GitHub и файлов, правка прямо на канвасе и экспорт в PDF и PowerPoint.
Обе фичи нацелены на то, чтобы сократить путь от идеи до работающего софта и упростить коммуникацию между дизайном и разработкой, один из самых давних стыков в индустрии.
