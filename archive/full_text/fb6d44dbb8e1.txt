Microsoft обратилась к Amazon за мощностями AWS, чтобы спасти GitHub от сбоев из-за ИИ
Рост ИИ-разработки заставляет GitHub искать вычислительные мощности у главного конкурента Microsoft — Amazon Web Services. Разбираем, почему это важно для разработчиков.
Новости TprogerMicrosoft арендует дополнительные облачные мощности у Amazon Web Services, чтобы GitHub справлялся с наплывом кода, который создают ИИ-инструменты. Если ваш репозиторий тормозил или был недоступен в последние месяцы — вот почему это происходит.
Business Insider со ссылкой на двух знакомых с планами людей сообщает: Microsoft, владелец GitHub, вынуждена арендовать дополнительные вычислительные мощности у Amazon Web Services. Официальный представитель Microsoft подтвердил лишь использование нескольких облачных провайдеров, но отказался комментировать участие Amazon. AWS — главный конкурент Microsoft Azure на рынке облаков, поэтому конкуренты вынуждены сотрудничать.
Ключевые выводы
- Microsoft планирует разместить часть нагрузки GitHub на Amazon Web Services из-за нехватки собственных мощностей.
- При сохранении текущего темпа число коммитов на платформе может вырасти с 1 млрд в 2025 году до 14 млрд в 2026 году.
- В 2026 году GitHub пережил десятки крупных сбоев.
- Microsoft намеревалась полностью перевести GitHub на Azure к 2027 году, но ИИ-бум ускорил потребность в дополнительных мощностях.
Что произошло
GitHub — популярная платформа для хранения и совместной работы над кодом. До поглощения Microsoft в 2018 году она в основном использовала собственные дата-центры. Корпорация планировала полностью перевести GitHub на Azure к 2027 году, но всплеск активности, связанный с разработкой на базе ИИ-агентов и ассистентов, заставил пересмотреть сроки и искать мультиоблачную схему.
Рост коммитов в 14 раз
По данным директора по операциям GitHub Кайла Дейгла, при сохранении текущего темпа число коммитов — записей об изменениях в коде — в 2026 году может достичь 14 млрд против 1 млрд в 2025. Microsoft, со своей стороны, прогнозирует общие капитальные затраты корпорации в $190 млрд в 2026 году, в основном на расширение дата-центров, но часть проектов откладывается.
Почему Microsoft идёт на сделку с AWS
Конкуренция между Azure и AWS не отменяет главную задачу — не допустить сбоев в работе сервиса. В 2026 году GitHub пережил десятки крупных сбоев. Кроме того, на рынке появляются альтернативы вроде Cursor и Claude Code, которые конкурируют с экосистемой GitHub и Copilot, и Microsoft приходится ускорять модернизацию инфраструктуры.
Конкуренция за разработчиков
GitHub больше не место для серьёзной работы, если он ежедневно выбрасывает вас на несколько часов.

Часто задаваемые вопросы
Почему GitHub не справляется собственными силами?
ИИ-инструменты резко увеличили объём кода, который разработчики создают и загружают на GitHub. Число коммитов может вырасти в 14 раз за год, а собственные дата-центры и планы по переходу на Azure не успевают за этим ростом. Облачные мощности (cloud capacity) — это вычислительные ресурсы, которые сервис может оперативно масштабировать.
Что такое мультиоблачная стратегия?
Это использование нескольких облачных провайдеров одновременно. В данном случае GitHub будет сочетать Azure и AWS, чтобы получить нужную мощность, масштабируемость и отказоустойчивость.
Какие ещё сделки на рынке ИИ-мощностей происходят?
В начале июня SpaceX и Google заключили контракт на $920 млн в месяц за вычислительные мощности с октября 2026 года по июнь 2029 года. Ранее облачное подразделение Google уже продавало мощности Anthropic.
Что делать разработчикам прямо сейчас?
Ничего радикального: следите за статус-страницей GitHub, не используйте сервис как единственное хранилище критичного кода и помните, что мультиоблачная схема должна повысить стабильность платформы в среднесрочной перспективе.
Выводы
Сделка Microsoft и Amazon — признак того, что спрос на ИИ-инфраструктуру превышает возможности даже крупнейших игроков. Для разработчиков главное — чтобы GitHub оставался стабильным: следите за статус-страницей сервиса и не используйте GitHub как единственное хранилище критичных данных. Первоисточник: Business Insider.
