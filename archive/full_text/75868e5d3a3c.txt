Google опубликовала график перехода к обязательной верификации разработчиков в системе Android. Первые изменения стартуют уже в текущем месяце.
июнь 2026: Google выпустила новый системный сервис для проверки регистрации разработчиков; до конца месяца он будет автоматически установлен на большинстве Android‑устройств;
июль 2026: Google откроет общедоступный API Android Developer ID Status для проверки регистрации пакетов, а также начнёт ограниченное тестирование API Android Developer Console для автоматической регистрации приложений;
август 2026: доступ к API Android Developer Console откроется для всех, также Google запустит новый тип бесплатных аккаунтов для энтузиастов и студентов и внедрит метод установки сторонних приложений от неверифицированных разработчиков;
30 сентября 2026 года: с этого дня обязательная верификация разработчиков вступит в силу для Бразилии, Индонезии, Сингапура и Таиланда. Требование будет применяться при установке приложений из каталога Google Play и магазинов партнёров: HONOR App Market, OPPO App Market (OPlus), Galaxy Store, Palm Store, V‑Appstore и GetApps;
2027 год: верификация будет действовать для всех приложений на сертифицированных устройствах Android по всему миру.
Напомним, согласно новым правилам, для публикации приложений в Google Play новым разработчикам придётся заплатить взнос в $25 и создать учётную запись в Android Developer Console. При регистрации нужно будет указать ФИО, адрес проживания, электронную почту, телефон и загрузить фото удостоверения личности. Организации нужно будет подтвердить сайт и предоставить международный идентификатор юридического лица (DUNS). После создания профиля разработчик должен добавить приложения и подтвердить авторство, указав имена пакетов и ключи цифровой подписи.
В начале года десятки организаций и независимых разработчиков подписали открытое письмо с критикой новой политики обязательной верификации. Среди 37 подписантов — Electronic Frontier Foundation, Free Software Foundation, Article 19, F-Droid, Fastmail и Vivaldi.
«Это расширяет полномочия Google по контролю за распространением приложений за пределы её рынка. Те разработчики, которые не хотят использовать сервисы Google, не должны быть вынуждены регистрироваться и подчиняться её решениям», — подчеркнули авторы.
