19 июня 2026 года состоялся релиз стабильной версии подсистемы инициализации и управления службами в Linux systemd 261.
Версию systemd 259 представили в декабре 2025 года. Сборка systemd 258 вышла в сентябре 2025 года. Версия systemd 257 стала доступна в декабре 2024 года. Решение systemd 256 опубликовали в июне 2024 года с run0 (на замену sudo), systemd-vpick, importctl и другими новыми функциями. Версию systemd 255 разработчики проекта выпустили в декабре 2023 года. В неё добавили компонент systemd-bsod, который обеспечивает поддержку ВSOD («синего экрана смерти») в Linux.
Среди ключевых изменений в systemd 261 можно выделить:
новая подсистема службы метаданных экземпляра IMDS. Она включает в себя новый systemd-imdsd, который делает службы IMDS доступными для локальных программ. Также имеется база данных оборудования для распознавания существующих публичных облаков по информации SMBIOS, таких как Amazon EC2, Microsoft Azure, Google Compute Engine, Oracle Cloud, Tencent Cloud, Hetzner и другие. Это обеспечивает удобный унифицированный способ доступа к метаданным виртуальных машин из systemd;
представлен storagectl как новый инструмент командной строки и интерфейс Varlink для унифицированного предоставления доступа к ресурсам хранения для использования в качестве управляемого пользовательского хранилища;
появился systemd-sysinstall, предоставляющий простой, современный текстовый установщик ОС. Он включает в себя возможности systemd по разметке диска, управлению учётными данными и другие функции управления системой, копируя ОС с временного загрузочного носителя, такого как USB-накопитель;
новая служба systemd-tpm2-swtpm.service для запуска IBM Software TPM "swtpm" для автоматического резервного копирования на системах, в которых отсутствует физический TPM, но где должна быть доступна функциональность TPM;
новый файл tmpfiles.d/root.conf, гарантирующий установку прав доступа к корневому каталогу на 0555;
PID1 теперь поддерживает возможности оркестратора Live Update Orchestrator "LUO" и Kernel Handover "KHO" ядра Linux;
менеджеры пользовательских сессий теперь поддерживают сохранение хранилищ файловых дескрипторов пользовательских модулей;
новая настройка модуля CPUSetPartition= позволяет настраивать тип раздела cgroup для набора ЦП, например, root, isolated или member, для служб systemd;
добавлена новая настройка RestrictFileSystemAccess=, использующая программу BPF LSM для ограничения выполнения только бинарных файлов, хранящихся в подписанной и проверенной файловой системе, защищенной DM-VERITY;
новые методы Varlink для интерфейса менеджера systemd для запроса завершения работы системы;
DefaultMemoryZSwapWriteback= — это новая настройка менеджера, устанавливающая значение по умолчанию для параметра отложенной записи Zswap в масштабе всей системы вместо существующей настройки для каждого модуля;
в отдельные бинарные файлы теперь встроено примечание о метаданных dlopen ELF.
