На сбор данных и проверку отчётов в Яндекс Метрике уходит огромное количество времени. Особенно, если у вас много клиентов и проектов. MCP Яндекс Метрики помогает упростить эти задачи. С его помощью можно запрашивать данные обычным текстом и быстро получать данные в удобном виде.
В этой статье разберём, что это за инструмент, как его подключить и эффективно использовать в работе.
Что такое MCP Яндекс Метрики
MCP (Model Context Protoco) — это инструмент, который позволяет работать с данными Яндекс Метрики через текстовые команды. Проще говоря, вы можете не искать вручную нужные отчёты в интерфейсе, а просто написать, что именно хотите увидеть.
Например, можно запросить:
все поисковые запросы по сайту;
количество прямых заходов;
источники трафика;
конверсии по каналам;
сравнение любых периодов;
посещение определённых страниц;
и многое другое.
+ MCP поддерживает разбивку данных по периодам.
Главное преимущество такого подхода — это экономия времени. Вместо того, чтобы каждый раз вручную собирать показатели по разным разделам, можно сформулировать один запрос и получить готовый ответ.
Где можно использовать MCP
Использовать MCP можно в разных редакторах кода и рабочих средах. Я использую OpenCode, но подойдут и другие решения того же типа: VS Code, Cursor, Cloud Code и аналогичные редакторы.
Суть от этого не меняется. Один раз настраиваете подключение, а дальше работаете с Метрикой через текстовые запросы.
Как подключить MCP Яндекс Метрики
Вначале находим репозиторий проекта и инструкции по установке. В описании есть файл с документацией, где перечислены возможности инструмента и показан код для подключения.
Шаг 1. Получите токен доступа
Чтобы инструмент мог работать с данными Метрики, нужен авторизационный токен. Для этого необходимо:
Войти в аккаунт Яндекс.
Создать приложение.
Указать его название.
При необходимости настроить дополнительные параметры.
Выдать права доступа.
Установите доступ на чтение. После этого вы получите токен, который нужно будет добавить в конфигурацию.
Шаг 2. Добавьте настройки в редактор
Далее открываете конфигурационный файл вашего редактора и вставляете туда блок подключения. Туда же — полученный токен.
Если вы уже создавали подключения ранее, новый блок просто добавляется рядом с ними.
Шаг 3. Перезапустите редактор
После сохранения настроек редактор лучше перезагрузить. Так он подхватит конфигурацию и подключил новый инструмент.
После первого запуска может появиться ошибка подключения к локальному серверу. Обычно это не критично и чаще всего связано с одной из двух причин:
ошибка в конфигурационном файле;
неправильно указан путь.
В таком случае нужно вернуться к настройкам и внимательно всё проверить. Если исправить путь или синтаксис, а затем снова перезапустить редактор, подключение обычно начинает работать нормально.
Как проверить, что всё подключено правильно
После настройки нужно убедиться, что система видит Яндекс Метрику и готова с ней работать. Для этого обычно передают ID счётчика и отправляют тестовый запрос на проверку подключения.
Если всё настроено корректно, инструмент покажет, что:
подключение активно;
счётчик доступен;
код установлен;
данные можно получать.
После этого можно переходить к полноценной работе.
Какие запросы можно отправлять в Метрику с помощью MCP
Да любые. Какие вам нужны для работы, такие и отправляйте. Разберу четыре самых частых случая.
Пример №1. Базовые показатели проекта за неделю
Вам нужно знать основные показатели по проекту за последние 7 дней. Запрашиваете:
показать посещения;
вывести отказы;
добавить глубину просмотра;
разбить всё по дням.
В ответ вы получите понятную сводку по каждому дню с основными показателями.
Пример №2. Источники трафика
Ещё один вариант запроса попросить показать топ источников трафика за определённый период и вывести конверсию по основным каналам.
Так можно быстро увидеть:
прямые заходы;
переходы из поисковых систем;
переходы из соцсетей;
показатели отказов;
длительность визитов.
Пример №3. Сравнение периодов
Один из самых полезных сценариев — сравнение показателей текущего месяца с прошлым. В этом случае можно быстро понять:
вырос ли трафик или снизился;
как изменилась доля отказов;
насколько изменилась конверсия;
есть ли заметные просадки.
Это помогает быстро оценить динамику проекта и заметить проблемы.
Пример №4. Поиск аномалий
Одна из самых полезных возможностей, когда нужно быстро понять, не произошло ли с проектом чего-то необычного:
резкого падения трафика;
всплеска отказов;
просадки по определённому каналу трафика;
странных изменений в поведении аудитории.
Запрос так и пишется: «Найди мне любые аномалии по…»
Как делать большие отчёты
MCP удобно использовать не только для быстрых запросов, но и для подготовки полноценных отчётов. Например, для клиента по проделанной работе.
Заранее составьте большой шаблон запроса, в котором указываются:
роль исполнителя, например аналитик данных;
ID счётчика Яндекс Метрики;
структура отчёта;
нужные разделы;
формат итогового файла (например, PDF).
В запросе можно сразу указать, что в отчёте должны быть:
количество визитов;
уникальные посетители сайта;
динамика по дням;
источники трафика;
сравнение с прошлым месяцем;
ГЕО;
итоги и выводы по трафику.
Почему удобно использовать шаблоны
Если вы часто работаете с аналитикой, есть смысл собрать собственную библиотеку готовых запросов. Например:
шаблоны для ежемесячных отчётов;
запросы для проверки рекламных кампаний;
шаблоны для оценки SEO-трафика;
шаблоны для оценки контекстной рекламы;
запросы для анализа конверсий;
сценарии для поиска аномалий.
Такой подход сильно ускоряет работу. Один раз продумали хорошие формулировки и потом просто используете их повторно.
Сколько времени занимает создание отчёта?
Небольшой отчёт может собраться за несколько минут. Если задача сложная, то полчаса или даже час.
Но здесь важен другой момент: вам не нужно тратить это время на ручную работу. Пока система собирает данные и оформляет документ, занимаетесь своими делами.
