Представляем MDN MCP server
Перевод статьи The MDN Team из MDN Blog. MDN MCP server даёт ИИ-агентам и IDE доступ к актуальной документации и данным о поддержке браузеров. Разбираем, как подключить и какую разницу он даёт в работе.
Перевод статьи The MDN Team из MDN Blog, оригинал: Introducing the MDN MCP server.
Мы рады анонсировать выпуск MDN MCP server. MCP (Model Context Protocol) — открытый стандарт, который позволяет ИИ-инструментам подключаться к внешним источникам данных. MDN MCP server использует этот протокол, чтобы перенести документацию MDN и данные о совместимости браузеров прямо в вашего ИИ-агента или IDE.

Зачем мы создали MDN MCP
Всё больше ИИ-инструментов интегрируется в рабочие процессы веб-разработки, но они могут выдавать устаревшую информацию о веб-платформе из-за обучающих данных и даты знания модели.
Например, LLM или агент для написания кода может не знать, что существует функция вроде @view-transition
CSS at-rule, или не знать, достигла ли она статуса Widely Available
в Baseline и безопасна ли для использования во всех браузерах.
MDN MCP даёт вашему агенту для написания кода доступ к точной и актуальной информации о веб-платформе. Также он упрощает доступ к последней документации, не покидая привычные инструменты.
Сервер сейчас находится в экспериментальном статусе. Подробности об обработке данных в этой фазе — в нашей заметке о приватности.

Какую разницу даёт MCP
Учитывая недетерминированную природу LLM и разнообразие доступных моделей, часто сложно сравнивать их поведение с включёнными или выключенными навыками, промптами, инструментами и MCP.
Мы протестировали Claude Code Opus 4.7 с MDN MCP и без него на нескольких функциях, недавно появившихся в Firefox 150 и 151, спрашивая, как использовать функции и какая у них поддержка браузеров. А именно:
- Как использовать CSS-функцию
light-dark()
для изображений и какие браузеры её поддерживают? - Как использовать CSS-псевдокласс
:buffering
и какие браузеры его поддерживают? - Как использовать атрибут
shadowrootslotassignment
на элементеи какие браузеры его поддерживают?
- Как использовать Web Serial API и какие браузеры его поддерживают?
Мы заметили определённые закономерности в результатах. В большинстве случаев заметки по использованию от Claude Code с MDN MCP и без него были сопоставимы. Часть ответов, использовавших MCP, была структурирована лучше и полнее. Например, заметки для CSS-функции light-dark()
также включали примеры с линейными градиентами, которые не были прямо упомянуты в вопросе, но тоже поддерживаются.
Когда дело дошло до информации о поддержке браузеров, победитель был очевиден: MDN MCP дал намного точнее и надёжнее результаты. Claude Code без MCP правильно определил поддержку браузеров только в одном случае — для псевдокласса :buffering
.
Например, Claude Code без MCP настаивал, что декларативный атрибут shadowrootslotassignment
поддерживается в Chrome 120 и Safari 18.3, возможно, путая его с опцией slotAssignment
метода Element.attachShadow()
. Но на самом деле Firefox 151 — первый браузер, в котором появилась поддержка этого атрибута.
Без MCP Claude Code также не дал никакой конкретной информации о поддержке браузеров для использования изображений в функции light-dark()
: «поддержка менее однородна, чем вариант с цветом», тогда как с MCP он выдал полную таблицу с указанием Firefox 150 и Chrome (за флагом) как поддерживающих браузеров.
Худший результат Claude Code без MCP показал на вопросе про Web Serial API. Firefox 151 получил поддержку Web Serial API в мае 2026 года. Однако Claude Code без MCP правильно упомянул браузеры на базе Chromium как поддерживающие эту функцию, но также настаивал, что в Firefox она:
Не реализовано (и не в планах — см. позицию Mozilla по стандартам: «вредно»).
С включённым MCP Claude Code правильно определил, что Firefox 151 поставляет поддержку Web Serial API, согласно примечаниям к выпуску.
Кроме того, мы заметили, что в наших тестах ответы с использованием MDN MCP были вдвое быстрее. Без MCP Claude Code приходилось загружать и парсить немало HTML-страниц, чтобы найти актуальную информацию, что занимало время, но даже тогда не давало точных результатов.

Часто задаваемые вопросы
Что такое MDN MCP server?
Это экспериментальный сервер по протоколу Model Context Protocol (MCP), который переносит документацию MDN и данные о совместимости браузеров прямо в ИИ-агент или IDE.
С какими инструментами работает MDN MCP server?
С любым MCP-совместимым клиентом: VS Code, Zed, Cursor, Claude Code, Codex CLI, Antigravity CLI и Claude Desktop.
Почему данные MDN актуальнее обучающих данных моделей?
У LLM есть дата знания, после которой они не знают о новых функциях веб-платформы. MDN MCP обращается к живой документации и данным о совместимости, поэтому информация актуальная.
Как подключить MDN MCP к Claude Code?
Выполните команду: claude mcp add --transport http mdn https://mcp.mdn.mozilla.net/
Насколько точнее ответы с MDN MCP?
В тестах на функциях Firefox 150 и 151 Claude Code с MDN MCP правильно определил поддержку браузеров там, где без MCP давал неточные или устаревшие данные, а ответы были вдвое быстрее.
