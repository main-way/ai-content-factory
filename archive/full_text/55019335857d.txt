Microsoft подтвердила наличие новой ошибки в Windows, из-за которой в запросах на удаление в корзине отображаются внутренние имена файлов вместо реальных, сообщает Neowin.
Компания Microsoft подтвердила, что после установки обновления Patch Tuesday за июнь (KB5094126) при использовании корзины могут возникать непредвиденные проблемы. По сути, при попытке удалить элемент из корзины в диалоговом окне подтверждения отображается внутреннее имя файла, а не фактическое название. Например, файл может называться abc.png, но в диалоговом окне подтверждения пользователя спросят, уверен ли он, что хочет безвозвратно удалить $Rxxxxx.png из корзины.
Правильное название отображается в списке корзины, и если восстановить файл, он вернётся с правильным названием. Эта проблема затрагивает практически все поддерживаемые версии клиентских и серверных операционных систем Windows, в том числе:
Клиент: Windows 11, версия 26H1; Windows 11, версия 25H2; Windows 11, версия 24H2; Windows 11, версия 23H2; Windows 10, версия 22H2; Windows 10 Enterprise LTSC 2021; Windows 10 Enterprise LTSC 2019; Windows 10 Enterprise LTSB 2016
Сервер: Windows Server 2025; Windows Server 2022; Windows Server 2019; Windows Server 2016; Windows Server 2012 R2; Windows Server 2012
На данный момент Microsoft работает над решением проблемы, которое будет выпущено в «будущем» обновлении Windows. Компания заявила, что коммерческие клиенты могут использовать временное решение уже сейчас, но для получения дополнительной информации нужно обратиться в службу поддержки Microsoft для бизнеса.
Последнее обновление Patch Tuesday, по всей видимости, вызвало не только эту ошибку: пользователи столкнулись с проблемами с доступом к OneDrive и Dropbox, блокировку восстановления BitLocker и появление BSOD.
