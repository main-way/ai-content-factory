18 июня 2026 года состоялся выпуск кроссплатформенного инструмента с открытым исходным кодом для создания скриншотов Flameshot 14.0. Исходный код проекта написан на C++ и опубликован на GitHub под лицензией GNU General Public License v3.0. Стабильная версия Flameshot 13.0 вышла в августе 2025 года. Выпуск Flameshot 0.1.0 произошёл в июне 2017 года.
Исходные сборки проекта доступны для Windows, Linux и macOS. В этом релизе значительно улучшена поддержка нескольких мониторов, добавлена поддержка дробного масштабирования, новые рабочие процессы захвата изображения, а также исправлены ошибки на всех платформах.
Использовать Flameshot достаточно просто: запустить приложение, выделить область, которую нужно захватить, добавить необходимые комментарии на экране и сохранить снимок на ПК, благодаря простому и понятному интерфейсу. Flameshot позволяет пользователям легко загружать свои скриншоты непосредственно в облако, чтобы легко поделиться ими с другими, например, загрузить изображение прямо в Imgur одним щелчком мыши и поделиться ссылкой.
Список изменений Flameshot 14.0:
единый подход к многомониторной/масштабируемой работе. Самое значительное изменение в этой версии заключается в том, что при создании скриншота пользователю будет предложено выбрать монитор для захвата изображения. Ранее в Linux и Windows виджет захвата отображался на всех мониторах. Это создаёт значительные проблемы по мере расширения поддерживаемых в Linux рабочих сред, и смешанное масштабирование становится все более распространённым;
в Linux основной способ создания скриншота — использование вызовов xdg-desktop-portal. Это единственный способ создания скриншотов в Wayland, и многие рабочие среды X11 также поддерживают его;
для пользователей минималистичных оконных менеджеров (i3, xmonad и так далее) без xdg-desktop-portal теперь доступна опция «Настройки → Общие → Использовать метод создания скриншотов в устаревшем режиме X11», которая обходит портал. Эта опция помечена как устаревшая, но гарантирует работу Flameshot в большем количестве сред;
меню в трее для выбора экрана. Если пользователи обычно запускают Flameshot из меню в трее, то теперь доступно новое выпадающее меню, которое позволяет напрямую выбрать монитор для захвата, минуя виджет выбора монитора;
захват активного монитора (X11 и Windows). При включении этой функции Flameshot полностью пропускает диалоговое окно выбора монитора и автоматически захватывает тот монитор, на котором находится курсор. Включите её в Настройках → Общие → «Захват активного монитора (пропуск выбора монитора)»;
глобальная комбинация клавиш для создания скриншотов в Windows Flameshot. В Windows теперь поддерживаются глобальные горячие клавиши «сделать скриншот», чтобы сделать скриншоты с помощью сочетаний клавиш, отличных от PrntScn. Опция также может определить, зарезервировала ли Windows клавишу PrtScn для «Ножниц», и предложить изменить этот параметр реестра или зарегистрировать Flameshot в качестве приложения для захвата экрана в системе;
метод захвата D-Bus (Linux). Новый метод D-Bus captureScreen() позволяет скриптам и внешним приложениям программно запускать захват графического интерфейса Flameshot, что ещё больше упрощает автоматизацию и интеграцию в Linux;
портативный двоичный режим (Windows). Flameshot теперь может хранить свою конфигурацию в файле flameshot.ini рядом с исполняемым файлом, а не в каталоге конфигурации пользователя. Это позволяет развертывать приложение на портативных устройствах/USB-накопителях. Управляется флагом CMake USE_PORTABLE_CONFIG (по умолчанию включён для Windows);
переработанная подкоманда Screen Подкоманда CLI screen команды flameshot была переработана для поддержки захвата каждого экрана отдельно. Новый флаг --edit/-e позволяет пользователю указать, какой экран нужно захватить. Пользователи могут использовать это для назначения пользовательских горячих клавиш для создания скриншотов определённых мониторов, чтобы обойти виджет выбора монитора;
поддержка Nix Flake. Теперь включён полный файл flake.nix для воспроизводимых сборок и разработки на NixOS. Используйте nix build, nix run или nix develop без дополнительных настроек.
