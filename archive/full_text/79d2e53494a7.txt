16 июня 2026 года состоялся выпуск проекта дистрибутива DietPi 10.5 для одноплатных ПК на базе архитектур ARM и RISC‑V, таких как Raspberry Pi, Orange Pi, NanoPi, BananaPi, BeagleBone Black, Rock64, Rock Pi, Quartz64, Pine64, Asus Tinker, Odroid и VisionFive 2. Версия проекта DietPi 9.0 вышла в январе 2024 года. Сборку DietPi 10.0 представили в январе 2026 года.
Дистрибутив DietPi построен на пакетной базе Debian и доступен в сборках для более чем 50 плат.
Решение DietPi также может применяться для создания компактных окружений для виртуальных машин и обычных ПК на базе архитектуры x86_64. Сборки для плат отличаются небольшим размером (в среднем 130 МБ) по сравнению с Raspberry Pi OS и Armbian. Инструментарий для сборки и сопровождения дистрибутива распространяется под лицензией GPLv2.
Проект DietPi оптимизирован для минимального потребления ресурсов и развивает несколько собственных утилит: интерфейс для установки приложений DietPi‑Software, конфигуратор DietPi‑Config, систему резервного копирования DietPi‑Backup, механизм ведения временных логов DietPi‑Ramlog (также поддерживается rsyslog), интерфейс для установки приоритетов выполнения процессов DietPi‑Services и систему доставки обновлений DietPi‑Update.
Утилиты в DietPi предоставляют консольный интерфейс пользователя с меню и диалогами на базе whiptail. Поддерживается режим полной автоматизации установки, позволяющий провести инсталляцию на платы без участия пользователя.
В версии DietPi 10.5 содержатся исправления багов и ошибок, выявленных в предыдущей версии, новый образ для Orange Pi 5B, упрощённая упаковка Linux, а также множество исправлений для одноплатных ПК.
С выпуска 10.0 для работы проекта в качестве минимальной поддерживаемой базовой системы требуется Debian 12 Bookworm. В свете этого системы, всё ещё работающие под управлением Debian 11 Bullseye, автоматически перемещаются в специальную ветку совместимости, при этом пользователям рекомендуется обновиться до Bookworm, прежде чем переходить на новую версию DietPi. Прямое обновление до DietPi 10.0 поддерживается только с DietPi 8.0 или более поздней версии. Более старые установки проекта должны быть обновлены поэтапно.
«Наряду с некоторыми другими изменениями в нашей системе сборки Armbian, это упрощает и делает более надёжным переключение между ветками ядра, а также позволяет нам безопасно устанавливать несколько веток ядра одновременно. Обновление DietPi будет означать обновление ядра, если оно ещё не выполнено, если только администратор явно не пометил его как приостановленное», — заявили разработчики проекта.
Основные улучшения в DietPi 10.5:
Raspberry Pi:
KMS/DRM теперь включён по умолчанию при установке графического приложения или среды рабочего стола через dietpi-software, и его можно переключать с помощью новой специальной записи в параметрах отображения dietpi-config. После удаления нескольких библиотек для устаревшего драйвера прошивки GPU его использование имеет практически только недостатки. Любое приложение, которое использовало его функции, например OpenMAX, DispmanX и MMAL, больше не может работать ни на одной поддерживаемой версии репозитория Raspberry Pi;
настройка камеры RPi в dietpi-config была обновлена для переключения на современный стек камер с API libcamera, а не на устаревший. Недавнее обновление Linux 6.18 удалило поддержку устаревшего стека камер MMAL, который также никогда не работал с Camera Module 3.
DietPi-Config: меню дисплея было переработано. Компонент dietpi-display теперь является стандартным инструментом для настройки разрешения/режима и поворота дисплея, удалены меню разрешения для конкретных устройств Raspberry Pi и Odroid, а также другие устаревшие и неработающие опции, а именно безмониторный режим RPi, поворот HDMI (теперь включён в dietpi-display) и шумоподавление блока питания. Как и раньше, когда это было возможно только на виртуальных машинах, теперь на любой системе с загрузчиком GRUB можно дополнительно настроить разрешение терминала GRUB и консоли ранней загрузки, до того, как будет применён видеорежим KMS, настроенный через dietpi-display.
DietPi-Software: в RustDesk Client после последнего релиза, в котором исправлена совместимость пакетов с пакетами Debian time64, была добавлена поддержка архитектуры ARMv7.
