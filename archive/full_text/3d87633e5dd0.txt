A2A Java SDK 1.0.0: стабильный релиз для агентных приложений на Java
Официальный Java SDK для протокола Agent2Agent достиг версии 1.0.0. Теперь Java-разработчики могут строить агентные приложения, которые взаимодействуют с агентами на других языках и фреймворках.
Новости TprogerJava-разработчики получили стабильный инструмент для создания агентных систем. A2A Java SDK 1.0.0 вышел в релиз и доступен в Maven Central с groupId org.a2aproject.sdk
. Для сборки клиента можно взять артефакт a2a-java-sdk-client
или импортировать a2a-java-sdk-bom
. Библиотека реализует протокол Agent2Agent (A2A) — открытый стандарт, управляемый Linux Foundation, для взаимодействия ИИ-агентов друг с другом.
Agent2Agent — это протокол, управляемый Linux Foundation (изначально представленный Google), который позволяет агентам на разных фреймворках и языках находить друг друга, делегировать задачи и обмениваться результатами. Для Java-экосистемы выпуск SDK версии 1.0.0 означает переход от экспериментальных сборок к инструменту, готовому к промышленной эксплуатации.
Релиз 1.0.0 появился на прошлой неделе, как сообщает InfoQ в очередном выпуске Java News Roundup. В него вошли исправления ошибок, обновление зависимостей и несколько новых возможностей: интеграционный тестовый набор на базе Quarkus для проверки совместимости разных SDK, а также расширение интерфейса A2AHttpResponse
и класса A2AClientHTTPError
для доступа к HTTP-заголовкам ответов.
Ключевые выводы
- A2A Java SDK 1.0.0 вышел в GA — первая стабильная версия для Java.
- Реализует протокол Agent2Agent для взаимодействия ИИ-агентов.
- Появился интеграционный тестовый набор на базе Quarkus.
- Доступен в Maven Central: org.a2aproject.sdk.
- Параллельно вышел A2A Jakarta 1.0.0.CR1 для Jakarta EE серверов.
Что вошло в релиз A2A Java SDK 1.0.0
- Новый интеграционный тестовый набор (ITK) на базе Quarkus для проверки кросс-SDK совместимости.
- Возможность получать HTTP-заголовки ответов через интерфейс A2AHttpResponse и класс A2AClientHTTPError.
- Исправления ошибок и обновление зависимостей.
- Готовность к использованию в production вместе с A2A Protocol Specification 1.0.
I am pleased to announce the release of A2A Java SDK 1.0.0.Final — our first GA release. The A2A Java SDK is the official Java implementation of the Agent2Agent (A2A) Protocol, an open standard that enables AI agents to communicate and collaborate regardless of underlying framework, language, or vendor.
Параллельно команда выпустила первый релиз-кандидат A2A Jakarta 1.0.0.CR1. Эта интеграция помогает запускать A2A-агентов внутри Jakarta EE серверов, таких как WildFly. В CR1 переименованы пакеты и артефакты, обновлён набор TCK для работы с A2A Java SDK 1.0.0 и настроен запуск CI на Windows.
Зачем это Java-разработчикам
SDK позволяет строить агентные приложения на Java без привязки к Python-стеку. Агенты могут общаться с агентами на Google ADK, LangGraph, CrewAI и других фреймворках через единый протокол. Это особенно важно для команд, где основная инфраструктура уже написана на Java. Рядом с A2A развивается протокол MCP: если MCP соединяет агента с инструментами, то A2A — агента с агентом.
Часто задаваемые вопросы
Что такое A2A?
Agent2Agent — открытый протокол для взаимодействия ИИ-агентов. Он позволяет агентам на разных фреймворках и языках делегировать друг другу задачи и обмениваться результатами.
Где взять A2A Java SDK 1.0.0?
Библиотека опубликована в Maven Central. GroupId — org.a2aproject.sdk. Исходники и примеры доступны в репозитории a2aproject/a2a-java на GitHub.
Зачем Java-разработчику этот SDK?
SDK позволяет строить агентные приложения на Java без привязки к Python-стеку. Агенты могут общаться с агентами на ADK, LangGraph, CrewAI и других фреймворках через единый протокол.
A2A Java SDK 1.0.0 делает Java полноценной платформой для агентных приложений: теперь можно подключить SDK из Maven Central и начать строить агентов, которые работают в одной экосистеме с Python- и TypeScript-решениями.
